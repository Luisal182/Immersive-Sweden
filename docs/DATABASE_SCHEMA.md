# Database Schema — Immersive Sweden

### Table: `organizations` (Supabase / PostgreSQL)

---

## Overview

The `organizations` table is the core of the Immersive Sweden platform. It stores all organizations displayed on the interactive map. The table is hosted on Supabase (PostgreSQL) and is accessed by both the public map application and the admin dashboard.

---

## Map & Filter Dependencies

These fields directly affect the public-facing map and filter system:

| Field                    | Used by                                             |
| ------------------------ | --------------------------------------------------- |
| `latitude` + `longitude` | Map marker position — **required to appear on map** |
| `technology`             | Technology dropdown filter                          |
| `industry`               | Industry dropdown filter                            |
| `organization_model`     | Organization Model dropdown filter                  |
| `organization_subtype`   | Other Organizations dropdown filter                 |
| `name`                   | Search field                                        |
| `city`                   | Search field                                        |

## Important Notes

**All fields except `id` can be null.** The database is designed to accept partial data — organizations can be added with only a name and city and completed later.

**Coordinates are required for map visibility.** An organization without `latitude` and `longitude` will be stored in the database and visible in the admin dashboard, but will not appear as a marker on the public map.

**Filter values are case sensitive.** The values in `technology`, `industry`, `organization_model` and `organization_subtype` must match exactly the accepted values listed above, otherwise the filter system will not work correctly for that organization.

**The `id` column is auto-generated.** Never include it in CSV imports or manual inserts.

---

## Adding New Columns

New columns can be added directly from the Admin Dashboard using the **+ Add Column** button. All new columns are created as `text` type with `null` as default. After adding a new column, a developer needs to update the `OrgRow` interface in `EditDrawer.tsx` to make it editable from the dashboard.

---

## Removing Columns

Columns can be removed from the Admin Dashboard using the **🗑 Drop Column** button. This requires double confirmation. **Dropping a column permanently deletes all data stored in it — this cannot be undone.**
